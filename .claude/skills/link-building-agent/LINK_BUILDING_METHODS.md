# Link Building Methoden im Detail

## 1. Guest Posting

### Beschreibung
Artikel auf anderen Websites veröffentlichen mit Link zurück zur eigenen Seite.

### Workflow

```
1. Websites finden, die Gastbeiträge akzeptieren
2. Content Guidelines lesen
3. Thema pitchen
4. Artikel schreiben
5. Link natürlich einbauen
6. Einreichen und Follow-up
```

### Websites finden

**Google-Suchen:**
```
"[nische]" + "gastbeitrag"
"[nische]" + "gastartikel einreichen"
"[nische]" + "schreiben sie für uns"
"[nische]" + "gastautor werden"
"[nische]" + "autoren gesucht"
inurl:gastbeitrag [nische]
intitle:"gastbeitrag" [nische]
```

### Pitch-Template

```
Betreff: Gastartikel-Idee: [Titel]

Hi [Name],

ich lese [Publikation] regelmäßig und schätze besonders
[spezifischer Aspekt].

Ich würde gerne einen Gastartikel beisteuern:

Titel: "[Titel]"

Gliederung:
1. [Punkt 1]
2. [Punkt 2]
3. [Punkt 3]

Der Artikel würde ca. [X] Wörter haben und [Mehrwert für Leser].

Zu mir: [1-2 Sätze Expertise]

Wäre das interessant?

Beste Grüße,
[Name]
```

### Link-Platzierung

```
✅ Gut:
- Natürlich im Content ("wie wir in [Artikel] zeigen...")
- Als weiterführende Ressource
- In der Autoren-Bio

❌ Vermeiden:
- Keyword-stuffed Anchor
- Erzwungener Link ohne Kontext
- Mehr als 1-2 Links pro Artikel
```

### Qualitätskriterien

```
Site muss haben:
- DA > 30 (idealerweise > 50)
- Echte Redaktion
- Regelmäßige Veröffentlichungen
- Thematische Relevanz
- Kein "Link Farm" Charakter
```

## 2. Broken Link Building

### Beschreibung
Defekte Links auf anderen Websites finden und als Ersatz vorschlagen.

### Workflow

```
1. Relevante Seiten mit Ressourcen-Links finden
2. Defekte Links identifizieren
3. Eigenen Content als Ersatz erstellen/identifizieren
4. Webmaster kontaktieren
5. Toten Link + Ersatz anbieten
```

### Tools

```
- Ahrefs: Site Explorer → Broken Links
- Check My Links (Chrome Extension)
- Dead Link Checker
- Screaming Frog
```

### Prospecting

**Methode 1: Competitor Dead Links**
```
1. Konkurrenten in Ahrefs eingeben
2. Site Explorer → Backlinks → Broken
3. Seiten exportieren, die auf tote Konkurrent-URLs verlinken
4. Eigenen Content als Ersatz anbieten
```

**Methode 2: Nischen-Ressourcen**
```
1. "[topic] ressourcen" / "[topic] nützliche links" suchen
2. Check My Links auf jeder Seite laufen lassen
3. Defekte Links dokumentieren
4. Passenden eigenen Content identifizieren
```

### Outreach-Template

```
Betreff: Defekter Link auf [Seitenname]

Hi [Name],

ich bin auf Ihre Seite "[Seitentitel]" gestoßen - tolle
Ressourcensammlung!

Mir ist aufgefallen, dass der Link zu "[Beschreibung]"
nicht mehr funktioniert:
[URL des defekten Links]

Falls Sie nach Ersatz suchen: Wir haben einen ähnlichen
Artikel zu [Thema]:
[Ihre URL]

[1 Satz was den Artikel gut macht]

Nur als Hinweis - vielleicht hilfreich!

Beste Grüße,
[Name]
```

### Erfolgsrate

```
Typical: 5-10% Link-Conversion
Optimiert: 10-20%

Faktoren:
- Qualität des Ersatz-Contents
- Relevanz zum Original
- Professionalität des Outreach
```

## 3. Resource Page Link Building

### Beschreibung
Seiten finden, die Ressourcen-Listen führen, und eigene Inhalte vorschlagen.

### Ressourcen-Seiten finden

**Google-Suchen:**
```
"[topic]" + "ressourcen"
"[topic]" + "nützliche links"
"[topic]" + "empfohlene websites"
"[topic]" + "linksammlung"
"[topic]" + "lesenswert"
inurl:ressourcen [topic]
inurl:links [topic]
intitle:ressourcen [topic]
```

### Qualifizierung

```
Checklist:
□ DA > 25?
□ Wird aktiv gepflegt (letzte Updates)?
□ Thematisch relevant?
□ Keine reine Link-Farm?
□ Kontaktmöglichkeit vorhanden?
```

### Outreach-Template

```
Betreff: Ressourcen-Vorschlag für [Seitenname]

Hi [Name],

ich habe Ihre Ressourcen-Seite zu [Thema] gefunden -
sehr hilfreich!

Ich wollte [Ihren Artikel / Ihr Tool] vorschlagen,
das gut dazu passen könnte:

[URL]

[2-3 Sätze warum es Mehrwert bietet]

Falls es passt, würde ich mich freuen, wenn Sie es
aufnehmen. Wenn nicht, kein Problem!

Beste Grüße,
[Name]
```

## 4. Skyscraper Technique

### Beschreibung
Besseren Content als die Konkurrenz erstellen und deren Backlinks übernehmen.

### Workflow

```
1. Content mit vielen Backlinks finden (Ahrefs Content Explorer)
2. Analysieren warum der Content gut rankt
3. Deutlich bessere Version erstellen
4. Alle verlinkenden Seiten kontaktieren
5. Auf besseren Content hinweisen
```

### "Besser" bedeutet:

```
- Aktueller (neuere Daten)
- Umfassender (mehr Tiefe)
- Besser gestaltet (Visuals, UX)
- Praktischer (Beispiele, Templates)
- Lesbarer (Struktur, Formatierung)
```

### Outreach-Template

```
Betreff: Aktualisierte Ressource zu [Thema]

Hi [Name],

ich habe gesehen, dass Sie in Ihrem Artikel "[Titel]"
auf [alten Artikel] verlinken.

Der Artikel ist von [Jahr] - inzwischen hat sich
einiges geändert.

Wir haben gerade einen aktualisierten Guide veröffentlicht:
[URL]

Was neu ist:
- [Update 1]
- [Update 2]
- [Update 3]

Falls Sie Ihren Artikel aktualisieren, könnte das eine
bessere Ressource für Ihre Leser sein.

Beste Grüße,
[Name]
```

## 5. Unlinked Brand Mentions

### Beschreibung
Erwähnungen der eigenen Marke finden, die nicht verlinkt sind, und um Link bitten.

### Mentions finden

**Tools:**
```
- Google Alerts (kostenlos)
- Mention.com
- Brand24
- Ahrefs Content Explorer
- BuzzSumo
```

**Google Alerts Setup:**
```
Alerts erstellen für:
- "[Markenname]"
- "[Produktname]"
- "[CEO Name]"
- "[Slogan oder Catchphrase]"
```

### Outreach-Template

```
Betreff: Danke für die Erwähnung!

Hi [Name],

ich bin gerade auf Ihren Artikel "[Titel]" gestoßen -
danke für die Erwähnung von [Marke]!

Eine kleine Bitte: Könnten Sie einen Link zu unserer
Website hinzufügen? Das würde Ihren Lesern helfen,
mehr zu erfahren.

[URL zur verlinkenden Seite]

Gerne revanchiere ich mich - haben Sie Content, den
wir teilen könnten?

Beste Grüße,
[Name]
```

### Erfolgsrate

```
Typical: 30-50% Conversion
(Höher als andere Methoden, da bereits positive Erwähnung)
```

## 6. HARO (Help a Reporter Out)

### Beschreibung
Journalisten-Anfragen beantworten und als Quelle zitiert werden.

### Workflow

```
1. Bei HARO anmelden (haro.help)
2. Relevante Kategorien wählen
3. Täglich Anfragen prüfen (3x am Tag)
4. Schnell und qualifiziert antworten
5. Link in Bio/Credentials
```

### Antwort-Template

```
Betreff: [Query ID] - [Kurze Zusammenfassung]

Hi [Name],

zur Ihrer Anfrage über [Thema]:

[Direkter Absatz mit Expertise/Quote - 2-3 Sätze]

[Optionaler zweiter Absatz mit weiterer Einordnung]

[Optionale Daten/Statistiken als Beleg]

---
[Ihr Name]
[Titel]
[Unternehmen]
[Website] ← Link hier!
[Kurzer Satz zur Expertise]
```

### Best Practices

```
✅ DO:
- Innerhalb von 1-2 Stunden antworten
- Direkt mit dem besten Zitat starten
- Expertise belegen
- Verfügbarkeit für Follow-up signalisieren

❌ DON'T:
- Zu lange warten
- Generische Antworten
- Zu werblich sein
- Anforderungen nicht lesen
```

## Risiko-Management

### White Hat (sicher)

```
✅ Guest Posting auf echten Publikationen
✅ Broken Link Building
✅ Resource Page Outreach
✅ HARO
✅ Unlinked Mentions
```

### Gray Hat (vorsichtig)

```
⚠️ PBN (Private Blog Networks)
⚠️ Link-Tausch
⚠️ Paid Links (ohne Kennzeichnung)
⚠️ Automatisiertes Outreach
```

### Black Hat (vermeiden)

```
❌ Link-Kauf
❌ Link-Spam
❌ Negative SEO
❌ Cloaking
❌ Doorway Pages
```
